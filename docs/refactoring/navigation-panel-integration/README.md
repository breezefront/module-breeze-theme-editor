## Проблеми

### Phase 1: HTML Integration ✅ ВИКОНАНО
Navigation кнопки не відкривають панелі, тому що **панелі відсутні в DOM**.

**Рішення:** Додати HTML розмітку панелей у `index.phtml` + CSS стилі.  
**Статус:** ✅ Виконано

### Phase 2: CSS Fix 🔴 ПОТРЕБУЄ ВИКОНАННЯ
Панель працює, але **з'являється справа замість зліва** (відрізняється від frontend UX).

**Проблеми:**
- ❌ Панель з'являється СПРАВА (має бути зліва)
- ❌ Iframe зменшується через width (має зсуватись через margin)
- ❌ Стилі змішані в `_admin-preview.less` (має бути окремий `panels/_panels.less`)
- ❌ Анімація через `right:` (має бути `transform:` для GPU-прискорення)

**Рішення:** Структурний рефакторинг CSS - створити окремий `panels/_panels.less` та виправити позиціонування.  
**Статус:** 🔴 Потребує виконання

### Phase 2: CSS Fix ⏳ ПОТРЕБУЄ ВИКОНАННЯ

**Швидкий план:**

1. **Створити** `view/adminhtml/web/css/source/panels/_panels.less` (10 хв)
   - Скопіювати логіку з frontend
   - Адаптувати для admin селекторів

2. **Очистити** `view/adminhtml/web/css/source/_admin-preview.less` (5 хв)
   - Видалити стилі панелей
   - Залишити тільки iframe preview
   - Додати margin-left для зсуву

3. **Додати імпорт** в `_module.less` (2 хв)
   - `@import 'panels/_panels.less';`

4. **Очистити кеш** та тестувати (5 хв)
   - Видалити `pub/static/adminhtml`
   - Видалити `var/view_preprocessed/css`
   - Hard refresh

5. **Перевірити результат:**
   - ✅ Панель зліва (не справа)
   - ✅ Preview зсувається вправо (не зменшується)
   - ✅ Transform анімація (плавна)
   - ✅ Responsive працює
